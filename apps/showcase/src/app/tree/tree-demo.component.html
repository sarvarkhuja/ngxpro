nxp-tree

Recursive, generic tree primitive. Provide a list of root values, a childrenHandler, and an optional [content] template — the component renders itself, recursively, with depth, expansion state, and a chevron toggle handled for you.

§ 01

Quickstart

Three pieces of furniture: a controller (state container), a tree node (data source), and an optional template (row renderer). All nodes start collapsed. State lives in a per-tree WeakMap.

Controller modes

Two directives share the [nxpTreeController] selector. The selector you reach for depends on whether you want to key expansion by component instance or by data value.

variant A · default
[nxpTreeController]
keyed by · component instance

    [value]="item"
    [childrenHandler]="fn" />

§ 07

Architecture

DI provider chain 

// each <nxp-tree> bumps NXP_TREE_LEVEL via useFactory
<nxp-tree>                       // level = 0
 ├── provides NXP_TREE_NODE
 ├── provides NXP_TREE_LEVEL = (parent ?? -1) + 1
 │
 └── <nxp-tree-item>
      ├── inject(NXP_TREE_LEVEL)         → 0
      ├── inject(NXP_TREE_CONTROLLER)    → controller
      ├── contentChildren(NXP_TREE_NODE) → children()
      │
      └── <nxp-tree-item-content>
           ├── inject(NxpTreeItemComponent)
           ├── inject(NXP_TREE_LEVEL)    → indent
           └── chevron + slot

<nxp-tree>                       // level = 1 (recursed)
 └── <nxp-tree-item> …

notes

  • non-reactive bridge
    The controller stores expansion in a plain WeakMap / Map, not a signal. Each tree-item re-reads it in ngDoCheck and writes the result into a local signal — same trick Taiga-ui/cdk uses.
  • content vs. view children
    Inner <nxp-tree> is content-projected into <nxp-tree-item>, so the item queries them via contentChildren(NXP_TREE_NODE) — not viewChildren.
  • expand animation
    Driven by <nxp-expand> from @ngxpro/cdk; the chevron rotates 90°, the row indents 16 px per level.
§ 08

Selection state

The tree component does not own selection — but it composes cleanly with nxpCheckbox, the nxpMapper pipe, and two pure helpers from the package: nxpComputeTreeChecked derives a parent's boolean | null state from its descendants; nxpToggleTreeChecked cascades a click down the subtree, returning a fresh Map so the pure pipe re-evaluates.

checkbox-tree.component.ts 
// Tri-state map: leaves only — parents are derived.
readonly map = signal<ReadonlyMap<Node, boolean | null>>(new Map());

readonly getValue = (n: Node, m: Map<Node, boolean | null>) =>
  nxpComputeTreeChecked(n, m, this.handler);

readonly onChecked = (n: Node, v: boolean) =>
  this.map.set(
    nxpToggleTreeChecked(n, v, this.map(), this.handler),
  );

// template ─────────────────────────────────────────
<div [nxpTreeController]="true">
  @for (item of data.children; track item) {{ '{' }}
    <nxp-tree
      [childrenHandler]="handler"
      [content]="content"
      [value]="item" />
  {{ '}' }}
</div>

<ng-template #content let-item>
  <label nxpLabel>
    <input
      nxpCheckbox
      type="checkbox"
      [ngModel]="item | nxpMapper: getValue : map()"
      (ngModelChange)="onChecked(item, $event)" />
    <small>{{ '{{' }} item.text {{ '}}' }}</small>
  </label>
</ng-template>
