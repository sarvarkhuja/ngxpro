nxp-notification

Single-toast component. Normally rendered for you by nxp-notification-host when you call NxpNotificationService.open(content, options). The rows below map 1:1 onto that second-argument options object.

Visual appearance / severity level. Drives the surface tint (status-*-pale), the accent icon color, the default icon, and the role / aria-live ARIA hooks — error / warning interrupt assertively, everything else announces politely. Size of the toast card. Controls padding, gap, icon size, and font scale. Screen edge the stack docks to. Toasts at the same position share one column; changing it mid-session opens a new column. Optional bold title rendered above the message content. Accepts an NxpDynamicContent — a string, TemplateRef, or component class. Default message body content. Accepts an NxpDynamicContent — a string, TemplateRef, or component class. The call-site open(message, …) argument overrides this. Either a Remix Icon class name (e.g. ri-rocket-2-line) or a function (appearance) => string that resolves one. Empty string falls back to the default icon for the current appearance. Whether to render a close (×) button on the toast. Auto-dismiss delay in milliseconds. Set to false to disable auto-dismissal — the toast stays until the user closes it or dismissAll() is called. Pauses on hover / focus and resumes on leave. Arbitrary typed payload carried on the notification (open<T>(content, data )). Read it back off the notifications() signal — handy for correlating a toast with the row that spawned it.

NxpNotificationService

Injectable dispatcher. The root-provided service every alert flows through.

Shows a toast and returns an Observable<void> that completes when it is fully dismissed — chain follow-up toasts off .subscribe({ complete }). Dismisses every active toast with the exit animation. Live, read-only signal of the active stack. Whether the stack is currently fanned out (hover state).

nxp-notification-host

Stack container. Render once at the root of your app — it subscribes to NxpNotificationService and positions / animates toasts as they come and go. No user-facing inputs of its own; set app-wide defaults via nxpNotificationOptionsProvider().
